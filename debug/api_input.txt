=== APIへの入力値 ===

【プロンプト】
【分類ルール】
- パーツ名には「カード」「スタッフ紹介」「l-imgR」「u-h2」「u-h3」などが含まれます。
- 同じスライド内のその他のテキストは、そのパーツに属する内容（本文・タイトル・画像・肩書き・名前・数字など）です。
- 複数の内容がある場合（例：カード2件）、リスト形式で出力してください。
- 表示順は**スライド上の視覚的な順番（上から下、左から右）**を維持してください。

【画像のプレースホルダー】
- 以下のようなテキストは画像が入ることを意味し、画像として扱う：
  - 「〇〇撮影いたします。」
  - 「写真」
  - 「画像を挿入」
  - 「イメージを挿入」
  - 「イラストを挿入」
  - 「図」
- **単独の「画像」「イメージ」「イラスト」という単語は、文脈によっては画像プレースホルダーではない場合があるため、慎重に判定する**
- 画像ファイル名は「{output_filename}_img{連番}.jpg」の形式を使用する
  - output_filenameはmain.pyで指定された値（例：first）
  - 連番は01から始まる2桁の数字（01, 02, ...）
- 画像タグは必ず以下の形式で出力する：
  <img src="images/under/{output_filename}_img{連番}.jpg" width="800" height="590" alt="">

【パーツ名の取り扱いルール】
- **パーツ名と完全一致する文字列（「カード」「スタッフ紹介」など）はHTML内に表示しないでください**
- **パーツ名は分類のための識別子としてのみ使用し、最終的なHTML出力には含めないでください。見出しタグ（h1〜h6）としても使用しないでください**
- **同じテキストは必ず1回だけ使用してください（見出し・本文・リストなど、どの要素でも）**

【見出しの階層構造とセクション】
- 見出しレベルに応じて、必ず対応するセクションで囲む
  - h2の要素は<section class="tall"><div class="container">で囲む
  - h3の要素は<section class="short">で囲む
  - h4の要素は<section class="x-short">で囲む
- 各セクションには、対応する見出しレベルの要素と、その見出しに関連するすべてのコンテンツを含める
  - tallセクションにはh2と、次のh2が出現するまでのすべてのコンテンツ
  - shortセクションにはh3と、次のh3が出現するまでのすべてのコンテンツ
  - x-shortセクションにはh4と、次のh4が出現するまでのすべてのコンテンツ
- **見出しは必ずコンテンツの内容に基づいて決定してください**
- **見出しがない場合は、sectionで囲まないでください**

【セクションの階層関係】
- <section class="tall">は最上位のセクション
- <div class="container">は必ずtallセクションの中に配置
- <section class="short">は必ずtallセクションの<div class="container">の中に配置
- <section class="x-short">は必ずshortセクションの中に配置
- セクションは必ず親セクションの中にネストする

【セクションの開始と終了ルール】
- 各セクションは次の同じレベルの見出しが出現するまで閉じない
- 見出し以外のコンテンツ（p、div、ul、ol、dlなど）は、次の同じレベルの見出しが出現するまで、直前のセクション内に配置する
- 例：h2セクションの場合
  ```html
  <section class="tall">
    <div class="container">
      <h2>見出し2</h2>
      <p>見出し2に関連するコンテンツ</p>
      <div class="list-card01">
        <!-- 見出し2に関連するコンテンツ -->
      </div>
      <!-- 次のh2が出現するまで、すべてのコンテンツをここに配置 -->
    </div>
  </section>
  <section class="tall">
    <div class="container">
      <h2>次の見出し2</h2>
      <!-- この見出し2に関連するコンテンツ -->
    </div>
  </section>
  ```
- shortセクション、x-shortセクションについても同様のルールを適用する

【正しいセクションの階層構造の例】
```html
<section class="tall">
  <div class="container">
    <h2>見出し2</h2>
    <section class="short">
      <h3>見出し3</h3>
      <section class="x-short">
        <h4>見出し4</h4>
      </section>
    </section>
  </div>
</section>
```

【見出しとして扱うべき内容】
- 以下の条件を満たす場合のみ、見出しとして扱う
  - そのコンテンツが明確なセクションの開始を示している
  - そのコンテンツの後に続く内容が、そのコンテンツに関連する説明や詳細である
  - そのコンテンツが階層構造の中で重要な位置づけにある
- 以下の場合は見出しとして扱わない
  - 単なるラベルや分類名
  - 補足的な説明や注釈
  - 本文の一部として扱うべき短いテキスト

【コンテンツの階層関係】
- ページ内で最初に出現するコンテンツは、必ずh2として扱う
- その後のコンテンツは、前のコンテンツの階層に応じて適切な見出しレベルを決定する
  - h2の直下のコンテンツはh3
  - h3の直下のコンテンツはh4
  - h4の直下のコンテンツはh5
  - h5の直下のコンテンツはh6
- **見出しレベルは必ずh2→h3→h4→h5→h6の順序で使用し、階層を飛ばしてはいけない**
- **例：h2の直下にh4やh5を配置してはいけない**

【HTML生成ルール】
- タイトルは <h3 class="l-ttl"> に入れるが、前後の見出しレベルに応じてh2〜h6を使い分けること
  - h2の直下のコンテンツならh3
  - h3の直下のコンテンツならh4
  - h4の直下のコンテンツならh5
  - h5の直下のコンテンツならh6
- **見出しレベルは必ず階層順序を守り、飛ばしてはいけない**
- 画像は <figure class="l-img"><img ...></figure> に入れる
- <span class="num"> には数字があった場合のみテキストを入れる（なければ省略）
- 説明文は <p> ... </p> に入れる
- 改行の扱い：
  - 改行が1つの場合（\n）：<br>タグに変換
  - 改行が2つ以上連続する場合（\n\n）：それぞれの段落を<p>タグで囲む
  例：
  入力：「段落1\n\n段落2\n\n段落3」
  出力：
  <p>段落1</p>
  <p>段落2</p>
  <p>段落3</p>

【最終確認ステップ】
1. 見出しレベルとsectionタグの確認
   - h2の数と<section class="tall"><div class="container">の数が一致しているか
   - h3は必ず<section class="short">で囲まれているか
   - h4は必ず<section class="x-short">で囲まれているか
   - sectionタグが正しくネストされているか（tall > container > short > x-short）
   - 見出しに関連するコンテンツが適切なセクション内に配置されているか
   - 次の同じレベルの見出しが出現するまでセクションが閉じられていないか
   - **見出しレベルが正しい階層順序（h2→h3→h4→h5→h6）で使用されているか**
   - **階層を飛ばした見出しレベル（h2の直下にh4やh5など）がないか**

2. その他の確認事項
   - パーツ名がHTMLに表示されていないか
   - 同じテキストが複数回使用されていないか
   - 画像タグの形式が正しいか
   - 改行の扱いが適切か

3. すべてのルールを満たしていることを確認してから出力

【2段階検証プロセス】
1. 第1段階：通常のHTML生成
   - プロンプトに従ってHTMLを生成
   - 出力をapi_output.txtに保存

2. 第2段階：見出しとsectionの検証
   - api_output.txtの内容を読み込み
   - 見出しレベルとsectionタグの構造を確認
   - 以下の点を重点的にチェック：
     * h2とtallセクションの対応関係
     * sectionタグの正しいネスト構造
     * 見出しレベルの適切な使用
   - 問題があれば修正
   - 修正後のHTMLを出力

【出力形式】
- 純粋なHTMLのみを出力してください
- コードブロックや説明文などの余分なテキストは一切出力しないでください
- 該当するパーツが見つからない場合は、<div class="no-parts">で囲んでください

【パーツ名】


- ボックス：<div class="box-style01">で囲み、タイトルとコンテンツを含める
  <div class="box-style01">
  <h2〜h6 class="box-ttl">タイトル</h2〜h6>
  コンテンツ
  </div>

  - コンテンツには、テキスト、画像、リストなど、様々な要素を含めることができる。

- u-h3：<h3 class="u-h3"></h3> に入れる

- レイアウト1：

      <span class="num"></span>
      <h2〜h6 class="l-ttl">タイトル</h2〜h6>
      <p>テキスト</p>
    </div>
  </div>
  - <span class="num">の値は「STEP1」「step01」「flow01」などの形式（英単語+数字）

    </div>
  </div>
  - <span class="num">の値は「STEP1」「step01」「flow01」などの形式（英単語+数字）

- l-imgR：<div class="l-imgR">...</div> で、画像とテキストを横並びにする。

    </div>
  </div>
