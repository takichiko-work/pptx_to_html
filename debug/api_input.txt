=== APIへの入力値 ===

【プロンプト】
【分類ルール】
- パーツ名には「カード」「スタッフ紹介」「l-imgR」「u-h2」「u-h3」などが含まれます。
- 同じスライド内のその他のテキストは、そのパーツに属する内容（本文・タイトル・画像・肩書き・名前・数字など）です。
- 複数の内容がある場合（例：カード2件）、リスト形式で出力してください。

【画像のプレースホルダー】
- 以下のようなテキストは画像が入ることを意味し、画像として扱う：
  - 「〇〇撮影いたします。」
  - 「写真」
  - 「画像を挿入」
  - 「イメージを挿入」
  - 「イラストを挿入」
  - 「図」
- **単独の「画像」「イメージ」「イラスト」という単語は、文脈によっては画像プレースホルダーではない場合があるため、慎重に判定する**
- 画像ファイル名は「{output_filename}_img{連番}.jpg」の形式を使用する
  - output_filenameはmain.pyで指定された値（例：first）
  - 連番は01から始まる2桁の数字（01, 02, ...）
- 画像タグは必ず以下の形式で出力する：
  <img src="images/under/{output_filename}_img{連番}.jpg" width="800" height="590" alt="">

【パーツ名の取り扱いルール】
- **パーツ名は分類のための識別子としてのみ使用し、HTML内に表示しないでください（見出しタグとしても使用しない）**
- **同じテキストは必ず1回だけ使用してください（見出し・本文・リストなど、どの要素でも）**
- **与えられたテキストのみを使用し、存在しないテキストを補わないでください**
- **ppt_texts.jsonの順番を大幅に変更しないでください（5行以上飛ばしてコンテンツを配置することを避ける）**

【見出しの階層構造とセクション】
- 見出しレベルに応じて、必ず対応するセクションで囲む
  - h2の要素は<section class="tall"><div class="container">で囲む
  - h3の要素は<section class="short">で囲む
  - h4の要素は<section class="x-short">で囲む
- 各セクションには、対応する見出しレベルの要素と、その見出しに関連するすべてのコンテンツを含める
  - tallセクションにはh2と、次のh2が出現するまでのすべてのコンテンツ
  - shortセクションにはh3と、次のh3が出現するまでのすべてのコンテンツ
  - x-shortセクションにはh4と、次のh4が出現するまでのすべてのコンテンツ
- **見出しは必ずコンテンツの内容に基づいて決定してください**
- **見出しがない場合は、sectionで囲まないでください**

【セクションの階層関係】
- <section class="tall">は最上位のセクション
- <div class="container">は必ずtallセクションの中に配置
- <section class="short">は必ずtallセクションの<div class="container">の中に配置
- <section class="x-short">は必ずshortセクションの中に配置
- セクションは必ず親セクションの中にネストする

【セクションの開始と終了ルール】
- 各セクションは次の同じレベルの見出しが出現するまで閉じない
- 見出し以外のコンテンツ（p、div、ul、ol、dlなど）は、次の同じレベルの見出しが出現するまで、直前のセクション内に配置する
- 例：h2セクションの場合
  ```html
  <section class="tall">
    <div class="container">
      <h2>見出し2</h2>
      <p>見出し2に関連するコンテンツ</p>
      <div class="list-card01">
        <!-- 見出し2に関連するコンテンツ -->
      </div>
      <!-- 次のh2が出現するまで、すべてのコンテンツをここに配置 -->
    </div>
  </section>
  <section class="tall">
    <div class="container">
      <h2>次の見出し2</h2>
      <!-- この見出し2に関連するコンテンツ -->
    </div>
  </section>
  ```
- shortセクション、x-shortセクションについても同様のルールを適用する

【正しいセクションの階層構造の例】
```html
<section class="tall">
  <div class="container">
    <h2>見出し2</h2>
    <section class="short">
      <h3>見出し3</h3>
      <section class="x-short">
        <h4>見出し4</h4>
      </section>
    </section>
  </div>
</section>
```

【見出しとして扱うべき内容】
- 以下の条件を満たす場合のみ、見出しとして扱う
  - そのコンテンツが明確なセクションの開始を示している
  - そのコンテンツの後に続く内容が、そのコンテンツに関連する説明や詳細である
  - そのコンテンツが階層構造の中で重要な位置づけにある
- 以下の場合は見出しとして扱わない
  - 単なるラベルや分類名
  - 補足的な説明や注釈
  - 本文の一部として扱うべき短いテキスト

【コンテンツの階層関係】
- ページ内で最初に出現するコンテンツは、必ずh2として扱う
- その後のコンテンツは、前のコンテンツの階層に応じて適切な見出しレベルを決定する
  - h2の直下のコンテンツはh3
  - h3の直下のコンテンツはh4
  - h4の直下のコンテンツはh5
  - h5の直下のコンテンツはh6
- **見出しレベルは必ずh2→h3→h4→h5→h6の順序で使用し、階層を飛ばしてはいけない**
- **例：h2の直下にh4やh5を配置してはいけない**

【HTML生成ルール】
- **見出しレベルは必ず階層順序を守り、飛ばしてはいけない**
- 画像は <figure class="l-img"><img ...></figure> に入れる
- <span class="num"> には数字があった場合のみテキストを入れる（なければ省略）
- 説明文は <p> ... </p> に入れる
- 改行の扱い：
  - 改行が1つの場合（\n）：<br>タグに変換
  - 改行が2つ以上連続する場合（\n\n）：それぞれの段落を<p>タグで囲む
  例：
  入力：「段落1\n\n段落2\n\n段落3」
  出力：
  <p>段落1</p>
  <p>段落2</p>
  <p>段落3</p>

【最終確認ステップ】
1. 見出しレベルとsectionタグの確認
   - h2の数と<section class="tall"><div class="container">の数が一致しているか
   - h3は必ず<section class="short">で囲まれているか
   - h4は必ず<section class="x-short">で囲まれているか
   - sectionタグが正しくネストされているか（tall > container > short > x-short）
   - 見出しに関連するコンテンツが適切なセクション内に配置されているか
   - 次の同じレベルの見出しが出現するまでセクションが閉じられていないか
   - **見出しレベルが正しい階層順序（h2→h3→h4→h5→h6）で使用されているか**
   - **階層を飛ばした見出しレベル（h2の直下にh4やh5など）がないか**

2. その他の確認事項
   - パーツ名がHTMLに表示されていないか
   - 同じテキストが複数回使用されていないか
   - 画像タグの形式が正しいか
   - 改行の扱いが適切か

3. すべてのルールを満たしていることを確認してから出力


【出力形式】
- 純粋なHTMLのみを出力してください
- コードブロックや説明文などの余分なテキストは一切出力しないでください
- 該当するパーツが見つからない場合は、<div class="no-parts">で囲んでください

【パーツ名】


- ボックス：<div class="box-style01">で囲み、タイトルとコンテンツを含める
  <div class="box-style01">
  <h2〜h6 class="box-ttl">タイトル</h2〜h6>
  コンテンツ
  </div>

  - コンテンツには、テキスト、画像、リストなど、様々な要素を含めることができる。

- u-h2：必ずh2である（文脈に関係なく絶対にh2）。<h2 class="u-h2"></h2> に入れる

- ドットリスト：
  - 改行（\n）で区切られた短いテキストの羅列は、ドットリストとして変換する。
    例：
    入力：「めまい、耳鳴り、頭痛\n冷え\n尿のトラブル\n高血圧、動悸」
    出力：
    <ul class="list-disc">
      <li>めまい、耳鳴り、頭痛</li>
      <li>冷え</li>
      <li>尿のトラブル</li>
      <li>高血圧、動悸</li>
    </ul>
  - パーツ名が「ドットリスト」と指定されている場合も、ドットリストとして変換する。

- ボタン：<a href="" class="btn-more">テキスト</a> で、リンクボタンを表示する。
  <a href="" class="btn-more">テキスト</a>

- u-h3：必ずh3である（文脈に関係なく絶対にh3）。<h3 class="u-h3"></h3> に入れる

- お問い合わせボックス：
  <div class="tel-layout">
    <h2 class="l-ttl">タイトル</h2>
    <p class="tac"><a href="tel:電話番号リンク" class="l-tel"><img src="images/share/icon_tel.svg" alt="TEL" width="25" height="31">電話番号</a></p>
    <p>テキスト</p>
  </div>

  - タイトルは必須
  - 電話番号リンクは必須（tel:から始まり、ハイフンなしの数字のみ）
  - テキストは任意

- l-imgR：<div class="l-imgR">...</div> で、画像とテキストを横並びにする。
  <div class="l-imgR">
    <figure class="l-img"><img src="images/under/{output_filename}_img{連番}.jpg" width="800" height="590" alt=""></figure>
    <div class="l-desc">
      <!-- テキストのみとは限らず、p、table、dlなども入る -->
      テキスト
    </div>
  </div>

- 定義：<dl class="dl-style01">で囲み、dtとddのペアを含める
  <dl class="dl-style01">
    <dt>テキスト</dt>
    <dd>テキスト</dd>
  </dl>

  - dtとddは必ずペアで存在する必要がある
    - dt一つにつきddは一つ
    - dtのみ、ddのみの状態は不可
    - ddはdtの説明や補足となる
    - 誤り例：
      <dl class="dl-style01">
        <dt>用語</dt>
      </dl>
    - 正しい例：
      <dl class="dl-style01">
        <dt>用語</dt>
        <dd>定義</dd>
      </dl>

  - dt、ddのペアが複数ある場合は、同じdl内で繰り返す
  - dd内ではドットリストを使用可能
    例：
    <dl class="dl-style01">
      <dt>所属学術団体（学会）</dt>
      <dd>
        <ul class="list-disc">
          <li>日本東洋医学会</li>
          <li>日本老年薬学会（老年薬学認定薬剤師）</li>
        </ul>
      </dd>
    </dl>

- テーブル：<table class="table-style01">で囲み、colgroup、tbody、tr、th、tdを含める
  - ヘッダーが行方向（左側）のテーブル：
  <table class="table-style01">
    <colgroup>
      <col style="width: 30%">
      <col style="width: 70%">
    </colgroup>
    <tbody>
      <tr>
        <th scope="row">見出し</th>
        <td>内容</td>
      </tr>
      <tr>
        <th scope="row">見出し2</th>
        <td>内容2</td>
      </tr>
    </tbody>
  </table>
  - 複数の行がある場合は<tr>を繰り返す
  - th要素には必ずscope="row"を指定する

  - ヘッダーが列方向（上側）のテーブル：
  <table class="table-style01">
    <colgroup>
      <col style="width:16%">
      <col style="width:12%">
      <col style="width:12%">
    </colgroup>
    <tbody>
      <tr>
        <th scope="col" class="tac">2024年12月<br>空席状況</th>
        <th scope="col" class="tac">月</th>
        <th scope="col" class="tac">火</th>
      </tr>
      <tr>
        <th scope="row" class="bg01">1日利用<br>9:30 – 15:00</th>
        <td>満員</td>
        <td>○</td>
      </tr>
    </tbody>
  </table>
  - 1行目（ヘッダー行）のth要素にはscope="col"を指定する
  - 2行目以降のth要素にはscope="row"を指定する
  - 列幅はcolgroupのcol要素で指定する
  - **thが2行以上ある場合もある**
  - **colspan、rowspanを使用してセルを結合する場合がある**
    - colspan：列方向のセル結合（例：colspan="2"で2列分結合）
    - rowspan：行方向のセル結合（例：rowspan="2"で2行分結合）

- アンカーリンク：<ul class="list-anchor-link">で囲み、各リンクを<li>で囲む
  例：
  <ul class="list-anchor-link">
    <li><a href="#link01">ケアマネージャーの方はこちら</a></li>
  </ul>

  - 要素が増えると、liを増やす。
  - 各リンクは<a href="#{アンカー名}">リンクテキスト</a>の形式で記述する。
  - アンカー名は「link01」「link02」「link03」のように、「link」＋2桁の数字の形式で、数字は連番とする。
  - **リンク先のidはセクションタグ（section）につける**
    - 例：<section class="tall" id="link01">
    - 例：<section class="short" id="link02">

【システムメッセージ】
あなたはパワーポイントスライドをHTML断片に変換するアシスタントです。
